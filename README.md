# LoadBrief

**Fine-tuning Llama 3 8B to turn athlete-monitoring narratives into structured load-management briefs.**

LoadBrief converts a free-text athlete monitoring summary (training load, heart-rate variability, wellness scores) into a structured, audience-conditioned load-management brief — declaring a risk level, an overreaching classification, specific recommendations, and an escalation trigger, written for an *athlete*, *coach*, or *sports scientist*.

The model, dataset, and a runnable demo are on Hugging Face:

- **Model:** https://huggingface.co/tylerhobbs/loadbrief *(link to be finalized)*
- **Dataset:** https://huggingface.co/datasets/tylerhobbs/loadbrief *(link to be finalized)*

> Built with Meta Llama 3. Model weights are subject to the [Meta Llama 3 license](https://llama.meta.com/llama3/license/).

## Motivation

Athlete monitoring produces rich time-series data, but turning that data into a clear, actionable interpretation is expert-dependent and inconsistent across audiences. LoadBrief explores whether a small, fine-tunable language model can apply established sports-science decision rules consistently and communicate them in audience-appropriate language — a structured generation task grounded in published criteria rather than open-ended advice.

## Key finding

Across a held-out test set, every trained configuration recovers the correct risk and overreaching classification roughly three-quarters of the time, with 97% of risk predictions within one class of ground truth and zero catastrophic misclassifications — while the untuned base model scores near zero on the structured fields.

The headline result concerns the training recipe itself: **supervised fine-tuning (SFT) alone matches or beats every GRPO variant within noise.** Across seven reinforcement-learning configurations (multiple seeds and learning rates) and two reward ablations, GRPO adds nothing measurable on top of SFT. When the training data is generated by deterministic rules and the RL reward is derived from those same rules, SFT already extracts essentially all the learnable signal. This is reported as an honest characterization of when the popular SFT→RL recipe is — and is not — worth its compute.

## How the data is generated

LoadBrief's training data is **synthetic but not LLM-generated.** A purpose-built simulator encodes four bodies of published sports science as quantitative thresholds — Gabbett (ACWR load zones), Plews et al. (HRV suppression), Meeusen et al. (overreaching criteria), and Hooper (wellness norms) — and applies them deterministically.

Generation runs *backward from the label*: each scenario's ground-truth classification is fixed first, then a realistic 28-day time series is sampled to match, and the same rules independently confirm the label holds. Because the data and its label come from the same literature-derived rules, every example is **correct by construction** with respect to the published criteria. No language model invents clinical content at any point.

The dataset spans 50,000 examples across 66 sports and three complexity tiers (clear-cut, moderate, and conflicting-signal scenarios), each rendered into three audience registers.

## Pipeline

```
Rule-based simulator
        |
LoadBrief-50K dataset  (labels correct by construction)
        |
Formatted pairs  (sft_train, grpo_train, sft_test)
        |
   SFT  -->  GRPO  (seeds + ablations)
        |
Generate completions  (run_baselines on the held-out test set)
        |
   +----------------------+----------------------+
   |                                             |
Rule-based metrics                          LLM judge
(reward, risk, overreaching)                (Gemini, 1-7 scale)
   |                                             |
   +-------------------+-------------------------+
                       |
            Paper tables and figures
```

## Method summary

**Base model:** Llama 3 8B Instruct, fine-tuned with LoRA so the full pipeline fits on a single commodity device.

**Training:** a supervised stage teaches the brief format and the three audience registers, followed by a Group Relative Policy Optimization (GRPO) stage that optimizes a five-component reward (correct risk declaration, recommendation specificity, overreaching terminology, escalation trigger, and length). GRPO is used over PPO (no value network, roughly half the memory) and over DPO (the task has a programmatic reward rather than human preference pairs).

**Evaluation:** every model is scored on the same held-out test set with (1) rule-based metrics — composite reward, exact-match accuracy on the structured labels, and within-one-class accuracy — and (2) an LLM-as-judge protocol rating clinical accuracy, actionability, and clarity. Model outputs are read relative to a reference reward ceiling computed from the ground-truth briefs themselves.

## Limitations

The model learns to apply established sports-science criteria consistently — not to predict real-world injury outcomes. Labels are correct with respect to the published rules, not validated against athlete outcomes. Validation against ratings from practicing sports scientists is identified as future work. The data is rule-generated, so the model inherits any gaps in the encoded criteria.
